# Исправления по безопасности

По результатам проведенного Code Review в приложении `task4/main.py` были внесены следующие исправления:

1. **Защита эндпоинтов (Authentication)**:
   - Добавлена зависимость `verify_api_key`. Теперь все методы изменения данных (`POST`, `PUT`, `DELETE`) требуют наличия корректного ключа в заголовке `x-api-key`.
   - При отсутствии или неверном ключе возвращается ошибка `401 Unauthorized`.

2. **Настройка CORS**:
   - Добавлен `CORSMiddleware`, что позволяет легально подключать фронтенд-приложения к API.

3. **Защита от XSS**:
   - Внедрена функция `sanitize_string`, которая удаляет HTML-теги из полей `name` и `description` перед сохранением в базу данных. Это предотвращает выполнение вредоносных скриптов при выводе данных на фронтенде.

4. **Валидация данных**:
   - Сохранены и усилены проверки Pydantic.

Все изменения проверены с помощью обновленных self-тестов в `main.py`.
