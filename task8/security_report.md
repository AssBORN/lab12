# Отчет по анализу безопасности (Code Review)

## Анализируемый код
Приложение: `task4/main.py` (FastAPI + SQLAlchemy + PostgreSQL).

## Найденные уязвимости и недостатки

### 1. Отсутствие аутентификации и авторизации (Критическое)
**Проблема**: Все эндпоинты (`POST`, `PUT`, `DELETE`) открыты для всех пользователей. Любой человек может создать, изменить или удалить любой товар в базе данных.
**Риск**: Потеря данных, вандализм, несанкционированное управление ассортиментом.
**Рекомендация**: Внедрить механизм JWT-токенов и разграничение прав доступа (например, только роль `ADMIN` или владелец товара может его редактировать/удалять).

### 2. Отсутствие защиты от CORS (Среднее)
**Проблема**: В приложении не настроены Cross-Origin Resource Sharing (CORS) политики.
**Риск**: При попытке обратиться к API с фронтенд-приложения, развернутого на другом домене/порту, браузер заблокирует запросы.
**Рекомендация**: Добавить `CORSMiddleware` из `fastapi.middleware.cors`.

### 3. Потенциал для DoS-атак (Низкое/Среднее)
**Проблема**: Отсутствует ограничение частоты запросов (Rate Limiting).
**Риск**: Злоумышленник может отправить тысячи запросов в секунду, что приведет к перегрузке БД и падению приложения.
**Рекомендация**: Использовать сторонние библиотеки, такие как `slowapi`, или настроить лимиты на уровне Nginx/API Gateway.

### 4. Валидация данных (Низкое)
**Проблема**: Хотя Pydantic используется, некоторые поля имеют базовые ограничения. Например, `name` и `description` могут содержать вредоносные скрипты (если фронтенд их выведет без экранирования).
**Риск**: XSS (на стороне клиента).
**Рекомендация**: Добавить очистку строк от HTML-тегов (sanitization) перед сохранением в БД.

### 5. Безопасность БД (Низкое)
**Проблема**: Использование простых паролей в `DATABASE_URL` (в `docker-compose.yml`).
**Риск**: Если секреты попадут в Git, база данных будет скомпрометирована.
**Рекомендация**: Использовать `.env` файлы и секреты Docker.

---

## Итог
Приложение безопасно с точки зрения SQL-инъекций (благодаря SQLAlchemy) и утечек соединений. Однако оно полностью незащищено от неавторизованного доступа, что делает его непригодным для использования в реальной среде без внедрения системы безопасности.
