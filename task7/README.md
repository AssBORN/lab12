# SQL-миграции для схемы базы данных маркетплейса

Данное задание содержит raw SQL миграции для развертывания и оптимизации базы данных, основанной на модели «Товар» из Задания 1, расширенной для поддержки связей предметной области.

## Схема данных

Реализованы следующие сущности и связи:
1. **Sellers (Продавцы)** $\rightarrow$ **Products (Товары)**: Связь «Один-ко-многим» (один продавец может выставить множество товаров).
2. **Products (Товары)** $\rightarrow$ **Reviews (Отзывы)**: Связь «Один-ко-многим» (один товар может иметь много отзывов).
3. **Products (Товары)** $\rightarrow$ **Orders (Заказы)**: Связь «Многие-ко-многим» через промежуточную таблицу `order_items` (один заказ может содержать разные товары, и один товар может быть в разных заказах).

## Файлы миграций

- `migrations/001_initial_schema.sql`: Создание всех базовых таблиц, определение первичных и внешних ключей, установка ограничений (CHECK) на цену и количество.
- `migrations/002_add_indices.sql`: Оптимизация производительности. Созданы индексы для:
    - Поиска товаров по имени.
    - Фильтрации товаров по продавцу.
    - Быстрого получения отзывов к товару.
    - Сортировки заказов по дате.

## Как применить миграции
Выполните SQL-скрипты в указанном порядке в вашей базе данных PostgreSQL:
```bash
psql -U username -d marketplace -f task7/migrations/001_initial_schema.sql
psql -U username -d marketplace -f task7/migrations/002_add_indices.sql
```
