# Объяснение логики SQL-запроса

Данный запрос формирует аналитический отчет по эффективности продавцов, используя механизм общих табличных выражений (CTE) для разделения расчетов финансовых метрик и расчета рейтингов.

## Логика работы запроса

1. **CTE `SellerStats`**:
   - **Фильтрация**: Сначала мы ограничиваем данные заказами за последние 30 дней и исключаем заказы со статусом `cancelled`.
   - **Сбор данных**: Объединяем таблицы `sellers`, `products`, `order_items` и `orders`. Это позволяет связать конкретного продавца с выручкой от его товаров в конкретных заказах.
   - **Метрики**:
     - `total_completed_orders`: Считаем количество уникальных ID заказов.
     - `total_revenue`: Суммируем произведение количества товара на цену покупки (`quantity * price_at_purchase`). Это дает выручку после применения скидок/купонов, так как `price_at_purchase` фиксирует фактическую стоимость в момент сделки.
     - `total_commission`: Рассчитываем комиссию платформы (в данном примере взято 12% от выручки).
     - `return_percentage`: Вычисляем долю заказов со статусом `returned` относительно общего числа заказов.
   - **Фильтр по объему**: С помощью `HAVING` оставляем только тех продавцов, у которых более 10 заказов.

2. **CTE `SellerRatings`**:
   - Поскольку отзывы привязаны к товарам, а не напрямую к заказам, мы создаем отдельную агрегацию. Считаем средний рейтинг всех товаров конкретного продавца.

3. **Финальный SELECT**:
   - Соединяем финансовые показатели и рейтинги через `LEFT JOIN` (так как у некоторых продавцов может не быть отзывов, и мы не хотим их терять).
   - Применяем `COALESCE` для замены `NULL` в рейтингах на `0`.
   - Сортируем результат по убыванию прибыли платформы (`total_commission`) и ограничиваем вывод первыми 20 записями.

## Рекомендации по индексации

Для ускорения этого тяжелого аналитического запроса рекомендуется добавить следующие индексы:

1. **`orders(order_date, status)`**: Составной индекс для быстрой фильтрации заказов за период и исключения отмененных.
2. **`order_items(product_id, order_id)`**: Для ускорения соединений (JOIN) между товарами и заказами.
3. **`products(seller_id)`**: Для быстрой группировки товаров по продавцам.
4. **`reviews(product_id, rating)`**: Для ускорения расчета среднего рейтинга.
