# Рефакторинг функции расчета стоимости

В данной папке представлен результат рефакторинга функции из `bad_main.py`.

## Список изменений и пояснения

1. **Осмысленные имена переменных**:
   - `p` $\rightarrow$ `price`, `q` $\rightarrow$ `quantity`, `s` $\rightarrow$ `seller_id`, `t` $\rightarrow$ `customer_type`, `r` $\rightarrow$ `region_id`, `cp` $\rightarrow$ `coupon_value`.
   - Промежуточные переменные `a, b, c, d, e, f` заменены на `total_price`, `discount_rate`, `tax_rate`, `commission_rate` и `final_price`. Это делает код самодокументированным.

2. **Устранение "магических чисел"**:
   - Все коэффициенты скидок, налогов и комиссий вынесены в словари-константы (`CUSTOMER_DISCOUNTS`, `REGIONAL_TAXES`, `SELLER_COMMISSIONS`). Теперь изменить ставку налога можно в одном месте, не затрагивая логику функции.

3. **Оптимизация логики (замена `if-elif-else` на `.get()`)**:
   - Вместо длинных цепочек условий для определения скидок и налогов теперь используются словари с методом `.get()`, что сокращает количество строк и улучшает читаемость.

4. **Добавление Type Hints**:
   - Добавлены аннотации типов для аргументов и возвращаемого значения (`float`, `int`, `Union[float, str]`), что облегчает поддержку кода и позволяет статическим анализаторам находить ошибки.

5. **Улучшение структуры и читаемости**:
   - Код разбит на логические этапы с помощью комментариев.
   - Добавлена строка документации (docstring) для описания назначения функции.

6. **Исправление логики расчета комиссии**:
   - Вместо вычисления комиссии как отдельной переменной `e`, расчет теперь интегрирован в формулу `final_price = total_price * (1 - commission_rate)`, что более лаконично.
